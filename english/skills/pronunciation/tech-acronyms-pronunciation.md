# Tech Acronyms Pronunciation

Texniki müsahibədə, meeting-lərdə və gündəlik iş söhbətlərində bu söz və abbreviation-ları düzgün tələffüz etmək sənin professional imicinə birbaşa təsir edir.

---

## 1. Ən Çox İşlədilən Acronym-lər

### Protokollar / Standartlar

| Acronym | Tələffüz | Nümunə cümlə |
|---------|---------|--------------|
| API | /ˌeɪ.piːˈaɪ/ → "ay-pee-eye" | "The API returns JSON." |
| REST | /rɛst/ → "rest" (bir söz) | "It's a REST API." |
| HTTP | /ˌeɪtʃ.tiːˌtiːˈpiː/ → "H-T-T-P" | "HTTP status 200." |
| HTTPS | "H-T-T-P-S" | "The site uses HTTPS." |
| SQL | /siːkwəl/ → "sequel" (ən yaygın) | "I use PostgreSQL." |
| PostgreSQL | "post-gres-Q-L" ya da "post-gres-sequel" | "We run Postgres." |
| MySQL | "my-S-Q-L" ya da "my-sequel" | "MySQL database." |
| NoSQL | "no-S-Q-L" ya da "no-sequel" | "It's a NoSQL database." |
| SSL | "S-S-L" | "SSL certificate expired." |
| TLS | "T-L-S" | "TLS 1.3 is required." |
| OAuth | /ˈəʊˌɔːθ/ → "oh-auth" | "OAuth 2.0 flow." |
| JWT | "J-W-T" (hərflər ayrı) | "Using JWT for auth." |
| CORS | /kɔːrz/ → "cores" (bir söz) | "CORS policy issue." |

## 3. "Hərflə" ya "söz" kimi?

**Qayda:** Acronym-lər ya "hərflər ayrı" ya da "bir söz" kimi tələffüz edilir.

| Tip | Nümunə |
|-----|--------|
| Hərflər ayrı | API, AWS, CI/CD, DNS, TLS |
| Bir söz kimi | CRUD, REST, ACID, YAML, JSON, regex |
| İkisi də qəbul | SQL (sequel / S-Q-L), nginx |

Əgər şübhə etsən — hərfləri ayrı de. Hər zaman aydın olur.

## 4. Cümlədə istifadə nümunələri

```
"The API endpoint returns a JSON response with a UUID field."
"We use JWT tokens over HTTPS with OAuth 2.0."
"The CRUD operations are handled by an ORM."
"Our CI/CD pipeline runs on GitHub Actions and deploys to GCP."
"The async handler uses a mutex to prevent race conditions."
"I need to update the YAML config and run kubectl apply."
"We have an SLA of 99.9% — any CORS issues would break it."
```
