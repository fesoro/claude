# Deploy / Ship / Release / Rollout / Launch — Tech Terminləri

## Səviyyə
B1-B2 (mütləq bilməli — tech interview)

---

## Əsas Cədvəl

| Söz | Kontekst | Nüans |
|-----|----------|-------|
| **deploy** | kodu serverdə qaldırmaq | texniki hərəkət |
| **ship** | müştəriyə çatdırmaq | iş slang-i, daha dinamik |
| **release** | yeni versiyanı təqdim etmək | versiya numreləməsi |
| **rollout** | yavaş-yavaş təqdim | mərhələli / %ilə |
| **launch** | böyük anons / start | product launch |

> **Qısa qayda:**
> - **deploy** = texniki
> - **ship** = bitirib göndərmək (slang)
> - **release** = rəsmi versiya
> - **rollout** = mərhələli deployment
> - **launch** = hadisə / başlanğıc

---

## 1. Deploy — Kodu Production-a Çıxarmaq

Texniki proses — server, environment, CI/CD.

### Nümunələr

- We **deploy** to production every Tuesday.
- The CI/CD pipeline **deploys** automatically.
- **Deploy** the new version.
- We need to **deploy** the fix ASAP.
- Blue-green **deployment** strategy.
- **Deployment** failed at stage 3.

### Derived

- deployment (isim)
- deployable
- re-deploy

---

## 2. Ship — Göndərmək / Təqdim Etmək (slang)

Startup / silikon vadisi mədəniyyətindən. "Tamamla və çıxar" mənasını daşıyır.

### Nümunələr

- We **ship** new features weekly.
- **Ship** it!
- Time to **ship** this.
- Did you **ship** the PR?
- The team **ships** fast.
- Move fast and **ship** things.

### İfadə

- **ship fast** → tez tez çıxarmaq
- **ship it** → artıq göndər!
- **shipping** → aktiv proses

---

## 3. Release — Versiyanın Təqdimi

Versiya numresi ilə əlaqəli. Daha rəsmi.

### Nümunələr

- We **released** version 2.1.
- The **release** notes are ready.
- **Release** date: March 5.
- Major / minor / patch **release**.
- Pre-**release** version.

### Expressions

- **release notes** → dəyişiklik qeydləri
- **release candidate (RC)** → son yoxlama versiyası
- **release branch** → release üçün git qolu
- **release cycle** → buraxılış dövrü

---

## 4. Rollout — Mərhələli / Tədricən Açılış

İstifadəçilərə tədricən təqdim. Risk azaldılır.

### Nümunələr

- **Rollout** to 10% of users first.
- Gradual **rollout** over two weeks.
- We'll **roll out** the feature tomorrow.
- **Roll out** the update region by region.
- Canary **rollout**.

### Forma

- **rollout** = isim
- **roll out** = fel (iki söz)

- The **rollout** was smooth. (isim)
- We'll **roll out** the new API. (fel)

---

## 5. Launch — Böyük Açılış / Başlanğıc

Marketinq + məhsul hadisəsi. Daha böyük miqyaslı.

### Nümunələr

- Product **launch** next month.
- We're **launching** a new service.
- Official **launch** date.
- Successful **launch**.
- Soft **launch** (sakit start).
- **Launch** event.

### Ship vs Launch fərqi

- **Ship** → hər feature, hər release
- **Launch** → böyük məhsul başlanğıcı

- We **ship** bug fixes weekly. ✓
- We **launched** the iOS app last year. ✓ (böyük)
- We're **launching** a new brand. ✓

---

## Hansını Nə Vaxt İşlədim?

### Daily standup

- "I'll **deploy** the fix after lunch." (texniki)
- "We **shipped** 3 features this sprint." (slang)
- "The **release** goes out Friday." (rəsmi)

### Planning meeting

- "The **rollout** starts Monday." (mərhələli)
- "We're planning the product **launch**." (böyük hadisə)

### Bug report

- "Let's **deploy** the hotfix." ✓
- "The new version will be **released** next week." ✓

---

## Tez Test

Hansı söz daha uyğun?

1. Product ______ is set for next quarter. (böyük məhsul başlanğıcı)
2. ______ the fix to production. (texniki hərəkət)
3. Can you ______ this PR today? (slang, bitir göndər)
4. The ______ notes list all changes.
5. We'll ______ this to 5% of users first. (mərhələli)

**Cavablar:** 1. launch, 2. deploy, 3. ship, 4. release, 5. roll out

## Interview Nümunələri

- "In my last project, we **deployed** daily and **released** major versions monthly."
- "We follow a gradual **rollout** strategy to minimize risk."
- "After the successful **launch**, we **shipped** weekly improvements."
- "Our CI/CD pipeline handles automatic **deployment** to staging."
