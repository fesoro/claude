# Let / Allow / Permit / Enable — İcazə Vermək Felləri

## Səviyyə
B1

---

## Niyə Çətindir?

Azərbaycancada "let", "allow", "permit" hamısı "icazə vermək", "enable" isə "mümkün etmək" / "aktivləşdirmək" mənasındadır. Amma ingilis dilindəki fərq həm rəsmilik, həm məna, həm də qrammatik strukturdadır: **let** to olmadan istifadə olunur (let me go — yox, let me to go!), qalanları isə fərqli kontekstlərə aiddir. Developer-lər üçün **enable** xüsusilə vacibdir — texniki yazılarda hər yerdə işlənir.

---

## Əsas Cədvəl

| Söz | Məna | Rəsmilik | Struktur | Passive? |
|-----|------|----------|---------|---------|
| **let** | icazə ver (informal) | qeyri-rəsmi | let + sb + V (to yox!) | ✗ yoxdur |
| **allow** | icazə ver (neytral) | neytral/rəsmi | allow + sb + to + V | ✓ var |
| **permit** | icazə ver (rəsmi) | çox rəsmi/legal | permit + sb + to + V | ✓ var |
| **enable** | mümkün et (texniki) | texniki/neytral | enable + sb + to + V | ✓ var |

> **Qısa qayda:**
> - **let** = informal icazə ("to" olmadan!)
> - **allow** = neytral icazə (rəsmi yazıda da işlənir)
> - **permit** = rəsmi/legal icazə (sign, sənəd)
> - **enable** = mümkün etmək (icazə yox — imkan, tech-də aktivləşdirmək)

---

## 1. Let — İnformal İcazə

Ən gündəlik söz. **Yalnız informal kontekstdə.** Passive forması yoxdur.

### Struktur (KRİTİK!)

- **let** + someone + V (infinitive **without** "to")
- Let me **go**. ✓ / ✗ Let me to go.

### Nümunələr

| İfadə / Kontekst | Nümunə | Azərbaycanca |
|------------------|--------|-------------|
| let me + V | "Let me check the logs." | nə edim icazə ver |
| let someone + V | "My manager let me work from home." | icazə verdi |
| let's + V | "Let's deploy after standup." | gəlin edək |
| let me know | "Let me know if you find the issue." | xəbər ver |
| let it go | "Let it go — we'll fix it next sprint." | burax |

### ⚠ Yanlış struktur

- ✗ "Let me **to** push this." (to yox!)
- ✓ "Let me **push** this."
- ✗ "Let's to deploy." (to yox!)
- ✓ "Let's **deploy**."

### Forma (dəyişmir!)

- let → let → let (bütün zamanlarda eynidir)
- "She **let** me merge it yesterday." ✓
- "I've **let** you down." ✓

### Nə zaman istifadə etmə

- ✗ Rəsmi yazıda: "The policy lets users access data." (allow daha uyğundur)
- ✓ Söhbətdə, Slack-də: "Let me handle this." ✓

---

## 2. Allow — Neytral İcazə

Həm gündəlik, həm rəsmi yazıda işlənir. Passive çox istifadə olunur.

### Struktur

- **allow** + sb + **to** + V → "They allow us to work remotely."
- **allow** + V-ing → "This config allows caching." (ümumi qayda)
- **be allowed to** + V → passive

### Nümunələr

| İfadə / Kontekst | Nümunə | Azərbaycanca |
|------------------|--------|-------------|
| allow + to | "The policy allows developers to push to staging." | icazə verir |
| is not allowed | "Force push is not allowed on main." | icazə verilmir |
| allow access | "Allow this role to access the API." | girişə icazə ver |
| allow traffic | "Allow traffic on port 443." | trafikə icazə ver |
| allow for | "Allow for some latency in integration tests." | nəzərə al |

### Let vs Allow (struktur fərqi!)

| | Let | Allow |
|-|-----|-------|
| Struktur | let + sb + **V** | allow + sb + **to + V** |
| Passive | ✗ yoxdur | ✓ "It is not allowed." |
| Rəsmilik | informal | neytral |

- "Let me **deploy**." ✓ (to yox)
- "Allow me **to deploy**." ✓ (to var)

---

## 3. Permit — Rəsmi / Legal İcazə

Qanuni, rəsmi sənəd, sign, policy. Gündəlik söhbətdə nadir.

### Struktur

- **permit** + sb + **to** + V
- **permit** + V-ing → "No smoking permitted."
- **be permitted to** + V → passive (çox işlənir)

### Nümunələr

| İfadə / Kontekst | Nümunə | Azərbaycanca |
|------------------|--------|-------------|
| not permitted | "No cameras permitted in the data center." | icazə verilmir (rəsmi) |
| is permitted to | "You are permitted to access production logs." | icazəniz var |
| permit as noun | "Work permit. Driving permit. Building permit." | icazə sənədi |
| permit access | "The policy does not permit external access." | icazə vermir |

### Allow vs Permit

| | Allow | Permit |
|-|-------|--------|
| Rəsmilik | neytral | çox rəsmi/legal |
| Kontekst | gündəlik + rəsmi | sign, sənəd, qanun |
| Nümunə | "Dogs not allowed." | "No cameras permitted." |

- "Dogs aren't **allowed** here." ✓ (gündəlik)
- "Photography is not **permitted**." ✓ (rəsmi sign)

---

## 4. Enable — İmkan Vermək (Texniki!)

**Ən vacib fərq:** enable = "icazə vermək" deyil — **"mümkün etmək", "imkan yaratmaq"**.
Tech kontekstdə: feature, setting, funksiya aktivləşdirmək.

### Struktur

- **enable** + N → "Enable dark mode." (feature aktivləşdir)
- **enable** + sb + **to** + V → "This tool enables us to debug faster."
- **enable** + V-ing → nadir

### Nümunələr

| İfadə / Kontekst | Nümunə | Azərbaycanca |
|------------------|--------|-------------|
| enable feature | "Enable the feature flag in production." | feature-i aktivləşdir |
| enable CORS | "Enable CORS for the API gateway." | CORS-u aktivləşdir |
| enable + to | "This SDK enables developers to build AI apps." | imkan verir |
| enable/disable | "The button is enabled / disabled." | aktiv / deaktiv |
| enable 2FA | "Enable two-factor authentication." | 2FA-nı aktivləşdir |
| enable logging | "Enable verbose logging for debugging." | loqu aktivləşdir |

### Allow vs Enable (ən çox qarışdırılan!)

| | Allow | Enable |
|-|-------|--------|
| Məna | icazə verir | mümkün edir / aktivləşdirir |
| Subyekt | bir şəxs/qayda icazə verir | bir şey/tool imkan yaradır |
| Nümunə | "Manager allowed me to leave." | "Tool enabled me to debug." |

- "The manager **allowed** me to work remotely." (icazə — subyekt insandır)
- "The VPN **enables** remote work." (imkan yaradır — subyekt tool/texnologiyadır)
- "**Allow** traffic on port 80." (firewall icazə verir)
- "**Enable** HTTPS in the config." (setting aktivləşdir)

### Tech kontekst (developer üçün ən vacib!)

- **Enable** feature flag → feature-i yandır
- **Enabled** / **Disabled** → UI button state
- **Enable** CORS, HTTPS, logging, monitoring
- "This microservice architecture **enables** independent deployments."
- "Caching **enables** the app to handle 10x more traffic."

## Struktur Müqayisəsi (KRİTİK!)

| Feil | Struktur | Passive? | Nümunə |
|------|---------|---------|--------|
| **let** | let + sb + **V** (to YOX!) | ✗ | "Let me **deploy**." |
| **allow** | allow + sb + **to + V** | ✓ | "Allow me **to deploy**." |
| **permit** | permit + sb + **to + V** | ✓ | "You are **permitted to** enter." |
| **enable** | enable + sb + **to + V** | ✓ | "This **enables** us **to scale**." |

**Yalnız "let" "to" olmadan işlənir!**

## Tez-tez Yanlış İşlənənlər

| ❌ Yanlış | ✅ Düzgün |
|-----------|-----------|
| "Let me to push this." | "Let me **push** this." |
| "Allow me go." | "Allow me **to go**." |
| "This enables to scale faster." | "This enables **us** to scale faster." |
| "The policy lets users to access..." | "The policy **allows** users to access..." |
| "Photography is not allowed." (on formal sign) | "Photography is not **permitted**." |
| "The manager enabled me to leave." | "The manager **allowed** me to leave." |
| "Enable me work from home." | "Allow me to work from home." / "**Enable** remote work." |
| "Let's to start the meeting." | "Let's **start** the meeting." |

## Məşq Tapşırıqları

### 1. Doğru sözu seçin (let / allow / permit / enable)

1. "______ me handle the on-call rotation this week." (informal, söhbət)
2. "Force push is not ______ on the main branch." (qayda, rəsmi)
3. "No photography ______ in the server room." (sign, çox rəsmi)
4. "This caching layer ______ the app to handle 50k RPS." (imkan yaradır)
5. "My CTO ______ me attend the conference." (informal, keçmiş)
6. "______ HTTPS in the Nginx config before deploying." (setting aktivləşdir)
7. "You are ______ to access staging but not production." (rəsmi icazə)
8. "The new API ______ developers to build apps without a backend." (imkan)
9. "They don't ______ junior devs to push directly to main." (qayda)
10. "______ me know if the build fails." (informal)

**Cavablar:** 1. Let, 2. allowed, 3. permitted, 4. enables, 5. let, 6. Enable, 7. permitted, 8. enables, 9. allow, 10. Let

### 2. Cümləni tamamlayın (düzgün struktur seçin)

1. "______ (Let / Allow) me ______ (to check / check) the server logs." (informal)
2. "The firewall ______ (allows / enables) traffic on port 443." (icazə — policy)
3. "This middleware ______ (allows / enables) us ______ (authenticate / to authenticate) requests." (imkan)
4. "Pets are not ______ (let / allowed) in the office." (qayda, passive)
5. "______ (Let's / Let us to) deploy after the tests pass." (informal təklif)
6. "No external access ______ (let / permitted) without VPN." (rəsmi)

**Cavablar:**
1. Let me check (to yox!) / Allow me to check
2. allows (policy icazə verir)
3. enables us to authenticate
4. allowed
5. Let's deploy (to yox!)
6. permitted
