# Mistake / Error / Fault / Bug / Issue — Səhv Sözləri

## Səviyyə
B1 (software engineering üçün xüsusilə vacib)

---

## Əsas Cədvəl

| Söz | Kontekst | Misal |
|-----|----------|-------|
| **mistake** | insan səhvi (qərar, yazı) | I made a **mistake** in the email. |
| **error** | texniki / rəsmi səhv | Syntax **error** in line 42. |
| **fault** | günah / qəbahət (kimi?) | It's my **fault**. |
| **bug** | proqramda səhv (dev konteksti) | This is a known **bug**. |
| **issue** | problem, məsələ (neytral) | I found an **issue** in the code. |
| **flaw** | qüsur, zəiflik (dizayn) | The plan has a **flaw**. |
| **glitch** | kiçik, keçici səhv | Just a small **glitch**. |

---

## 1. Mistake — İnsan Səhvi

İnsanın etdiyi səhv — qərar, əməl, yazı.

### Struktur: make a mistake (do yox!)

- I **made a mistake** yesterday.
- Don't **make mistakes** again.
- Everyone **makes mistakes**.

> ⚠ "do a mistake" YANLIŞdır. Həmişə **make a mistake**.

### Nümunələr

- I **made a mistake** in the report.
- It was a big **mistake**.
- Learning from **mistakes** is important.
- Sorry, my **mistake**.
- Don't worry, it's an honest **mistake**.

### Collocations

- make a **mistake** / commit a **mistake** (formal)
- a silly / careless / honest / simple **mistake**
- learn from **mistakes**
- by **mistake** (səhvən)
- **mistake** A **for** B (A-nı B zənn etmək)

---

## 2. Error — Texniki / Rəsmi Səhv

Daha formal — kompüter səhvi, mühasibat, test.

### Nümunələr

- There is an **error** in the code.
- **Error 404**: Page not found.
- A **system error** occurred.
- The test has 3 **errors**.
- This is a **rounding error**.

### Collocations

- **error** message / log / code
- syntax / runtime / logic **error**
- human **error** (insan səhvi, formal)
- a minor / major / critical **error**
- throw an **error** (proqram termini)

### Mistake vs Error — Fərq

| Mistake (adi) | Error (rəsmi / texniki) |
|---------------|-------------------------|
| I made a **mistake** in my essay. | The compiler showed an **error**. |
| It was a silly **mistake**. | A critical **error** occurred. |
| Everyone makes **mistakes**. | Minimize human **errors**. |

İnsan və ya elmi: bəzən eyni, amma `error` daha ciddi / texniki.

---

## 3. Fault — Günah / Qəbahət

"Kim günahkardır?" sualına cavab verir.

### Struktur

- It's my / his / her **fault**.
- It's not my **fault**!

### Nümunələr

- It's my **fault**, I'm sorry.
- Whose **fault** is it?
- It's not your **fault**.
- The accident was the driver's **fault**.

### Fault ilə qurğu qüsuru

Texniki nöqsan da bildirir:

- There's a **fault** in the engine.
- We need to find the **fault** in the system.
- A **fault** in the wiring.

### Mistake vs Fault

- **mistake** = yanlış qərar / əməl
- **fault** = kimin günahı

Nümunə:
- I **made a mistake** — it was my **fault**. (səhv etdim, mən günahkaram)

---

## 4. Bug — Proqramda Səhv

Software engineering konteksti.

### Nümunələr

- I found a **bug** in the login page.
- This is a known **bug**.
- We fixed the **bug** yesterday.
- The release had several **bugs**.

### Collocations

- fix / find / report a **bug**
- a critical / minor / blocker **bug**
- **bug** report / tracker / fix
- **bug**-free code
- edge case **bug**

### Bug vs Error vs Issue

- **bug** = proqramda davranış səhvi
- **error** = message görünən (exception)
- **issue** = ümumi problem / məsələ

---

## 5. Issue — Ümumi Problem / Məsələ

Neytral söz. Problem adlandırarkən "soft" variant.

### Nümunələr

- We have an **issue** with the database.
- Please report any **issues**.
- There's a minor **issue** in the code.
- I'd like to raise an **issue**. (müzakirə mövzusu)

### Collocations

- raise / address / resolve an **issue**
- a major / minor / technical **issue**
- **issue** tracker (GitHub issues)
- open / close an **issue**

### Problem vs Issue vs Bug

| Söz | Ton | Nümunə |
|-----|-----|--------|
| **problem** | güclü, neqativ | We have a serious **problem**. |
| **issue** | neytral, peşəkar | We have an **issue** to discuss. |
| **bug** | texniki | I found a **bug**. |

> İş yerində "issue" daha peşəkar görünür. "Problem" çətinlik deməkdir.

---

## 6. Flaw — Qüsur (Dizayn / Xarakter)

Daha çox dizayn, plan, xarakter qüsuru.

- There's a **flaw** in our strategy.
- A design **flaw**.
- A fatal **flaw**.
- A character **flaw**.

### Flaw vs Bug

- **bug** = konkret proqram səhvi
- **flaw** = dizayn qüsuru (daha geniş)

---

## 7. Glitch — Kiçik / Keçici Səhv

Kiçik, keçici texniki problem.

- A small **glitch** in the system.
- Sorry, there was a **glitch**.
- The video had a **glitch**.

Adətən özündən keçir, böyük problem deyil.

---

## 8. Defect / Fault / Defect

### Defect — istehsal / material qüsuru

- A manufacturing **defect**.
- The product has a **defect**.

Daha çox istehsal məhsullarında işlənir.

---

## 9. Keyword Müqayisəsi

### "I made a mistake" / "There was an error" / "It's a bug"

| Kontekst | Düzgün söz |
|----------|------------|
| Emaildə səhv yazdım | **mistake** |
| Sistemdə xəta | **error** |
| Kod səhv işləyir | **bug** |
| Problem olmuş ola bilər | **issue** |
| Dizaynda yanlışlıq | **flaw** |
| Kiçik keçici problem | **glitch** |
| Günahkar kimdir? | **fault** |

---

## 10. İş / Müsahibə Kontekstində

### Müsahibə — səhvdən danışarkən (STAR)

- "I **made a mistake** — I deployed without testing."
- "The root cause was an **error** in the config."
- "There was a **bug** in the auth flow."
- "We found several **issues** during QA."
- "The design had a **flaw** we missed initially."

### Problem həll etmə

- "Let's address this **issue** first."
- "The **error** message was misleading."
- "I reported the **bug** immediately."
- "Who's **fault** is this?" → "Let's focus on the fix, not blame."

### Retrospective

- "We made several **mistakes**, but we learned."
- "The **root cause** was a design **flaw**."
- "The **errors** in the log led us to the issue."

---

## 11. Xüsusi İfadələr

### Mistake ilə

- By **mistake** — səhvən (I sent the email **by mistake**.)
- Make no **mistake** — heç şübhə yoxdur (emfatik)
- Mistake A for B — A-nı B zənn etmək (I **mistook** him for his brother.)
- Learn from your **mistakes** — səhvlərdən öyrən

### Fault ilə

- To a **fault** — həddindən artıq (kind to a fault)
- Find **fault with** — tənqid etmək
- It's not my **fault** — mənim günahım deyil

### Bug ilə

- Bug-free — səhvsiz
- Edge case **bug** — nadir şərait səhvi
- Regression — əvvəl düzəldilmiş bug qayıdır

---

## 12. Tez-tez Edilən Səhvlər

| Səhv | Düzgün | İzah |
|------|--------|------|
| I did a mistake. | I **made a mistake**. | make a mistake |
| It was a big error (insan səhvi). | It was a big **mistake**. | insan qərarı → mistake |
| The code has an mistake. | The code has a **bug** / an **error**. | kod → bug/error |
| Whose mistake is it? (kim günahkar?) | Whose **fault** is it? | günah → fault |
| It's not my error. (kim günahkar?) | It's not my **fault**. | günah → fault |
| I found a fault in the code. | I found a **bug** / an **issue** in the code. | kod → bug |
| We have a big issue (böyük fəlakət). | We have a big **problem**. | güclü → problem |
| The system has a bug in design. | The system has a **flaw** in design. / a design **flaw**. | dizayn → flaw |
| Advices were helpful. | **Advice** (uncount) → **The advice was** helpful. | advice uncount |

## Məşq Tapşırıqları

### 1. Düzgün sözü seçin

1. I ___ a mistake in my report.
2. The login page has a ___ — users can't submit the form.
3. Whose ___ is it that we missed the deadline?
4. There was a small ___ in the video, but it's fixed now.
5. The plan has a ___ — we didn't consider edge cases.
6. Syntax ___ in line 15.
7. Please report any ___ in the issue tracker.
8. The team made several ___ this sprint, but we learned.

**Cavablar:** 1-made, 2-bug, 3-fault, 4-glitch, 5-flaw, 6-error, 7-issues, 8-mistakes

### 2. mistake / error / fault / bug / issue seçin

1. Sorry, my ___. (səhv, üzr)
2. The test showed 3 ___. (texniki)
3. It's not my ___. (günah)
4. I found a ___ in production. (kod)
5. We have some ___ to discuss. (məsələ)

**Cavablar:** 1-mistake, 2-errors, 3-fault, 4-bug, 5-issues

### 3. Tərcümə edin

1. Hesabatda səhv etdim. → I **made a mistake** in the report.
2. Sistemdə xəta baş verdi. → An **error** occurred in the system.
3. Bu mənim günahım deyil. → It's not my **fault**.
4. Kodda bir bug tapdım. → I found a **bug** in the code.
5. Müzakirə etmək üçün bir məsələ var. → There's an **issue** to discuss.
