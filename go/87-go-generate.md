# go generate: Kod Generasiyası (Senior)

## İcmal

`go generate` — Go fayllarındakı xüsusi şərh direktivi ilə xarici alətləri çalışdırır. Özü heç nə generasiya etmir — `stringer`, `mockery`, `sqlc`, `protoc` kimi alətlərə körpü rolunu oynayır. Source faylından tam tip-təhlükəsiz kod yaradır, boilerplate yazma yükünü aradan qaldırır.

## Niyə Vacibdir

- `stringer` — `iota` enum-ları üçün `String()` metodu avtomatik yaradır
- `mockery` — interface-dən mock yaradır — əl ilə mock yazmaq biter
- `sqlc` — SQL sorgularından tip-təhlükəsiz Go kodu yaradır
- `protoc` — `.proto` faylından gRPC server/client kodu yaradır
- Boilerplate sıfırlanır — generasiya edilmiş kod dəyişdirilmir, yalnız yenidən generate edilir

## Əsas Anlayışlar

**Direktiv formatı:**
```go
//go:generate <komanda> [arqumentlər]
```

Şərt: `//go:generate` — ikiqat slash, boşluq yoxdur, fayl başında yox — istənilən yerdə.

**Çalışdırma:**
```bash
go generate ./...        # bütün paket
go generate ./internal/... # konkret qovluq
go generate             # cari paket
```

**`$GOFILE`, `$GOPACKAGE` dəyişənlər:**
- `$GOFILE` — direktiv olan faylın adı
- `$GOPACKAGE` — paketin adı
- Komanda içində istifadə edilə bilər

## Praktik Baxış

**Ən çox istifadə olunan alətlər:**
- `stringer` — enum String() metodu
- `mockery` — interface mock
- `sqlc` — SQL → Go kodu
- `wire` — Dependency Injection kod generasiyası
- `oapi-codegen` — OpenAPI spec → Go server kodu

**Qaydalar:**
- Generasiya edilmiş faylları Git-ə commit et — CI-da `go generate` çalışdırma
- Fayl başında `// Code generated by ... DO NOT EDIT.` yaz
- `generate.go` faylı yaradıb orada direktivləri topla

**Common mistakes:**
- `go generate` çalışdırmadan pull request göndərmək — köhnə mock ilə PR
- Generasiya edilmiş faylı əl ilə dəyişmək — yenidən generate olunduqda itir
- `-run` flagsız çalışdırmaq — bütün generate çalışır

## Nümunələr

### Nümunə 1: stringer — enum String() metodu

```go
// status.go
package order

//go:generate stringer -type=Status -output=status_string.go

type Status int

const (
    StatusPending Status = iota
    StatusPaid
    StatusShipped
    StatusDelivered
    StatusCancelled
)
```

```bash
# Quraşdırma
go install golang.org/x/tools/cmd/stringer@latest

# Çalışdır
go generate ./...
```

```go
// status_string.go — avtomatik yaranır, dəyişdirmə!
// Code generated by "stringer -type=Status -output=status_string.go"; DO NOT EDIT.

package order

import "strconv"

func _() {
    // An "invalid array index" compiler error signifies that the constant values have changed.
    var x [[1]struct{}
    _ = x[StatusPending-0]
    _ = x[StatusPaid-1]
    // ...
}

const _Status_name = "StatusPendingStatusPaidStatusShippedStatusDeliveredStatusCancelled"

var _Status_index = [...]uint8{0, 13, 23, 36, 50, 65}

func (i Status) String() string {
    if i < 0 || i >= Status(len(_Status_index)-1) {
        return "Status(" + strconv.FormatInt(int64(i), 10) + ")"
    }
    return _Status_name[_Status_index[i]:_Status_index[i+1]]
}
```

```go
// İstifadə
order := Order{Status: StatusPaid}
fmt.Println(order.Status) // "StatusPaid"
// Əl ilə String() yazmaq lazım deyil!
```

### Nümunə 2: mockery — interface-dən mock yaratmaq

```go
// repository.go
package domain

//go:generate mockery --name=UserRepository --output=../mocks --outpkg=mocks

type UserRepository interface {
    FindByID(ctx context.Context, id int64) (*User, error)
    FindByEmail(ctx context.Context, email string) (*User, error)
    Create(ctx context.Context, user *User) error
    Update(ctx context.Context, user *User) error
    Delete(ctx context.Context, id int64) error
}
```

```bash
# Quraşdırma
go install github.com/vektra/mockery/v2@latest

# Çalışdır — mocks/MockUserRepository.go yaranır
go generate ./internal/domain/...
```

```go
// Test-də istifadə — əl ilə mock yazmağa ehtiyac yoxdur
func TestUserUsecase_GetUser(t *testing.T) {
    mockRepo := mocks.NewUserRepository(t)

    // Mock davranışı müəyyən et
    mockRepo.On("FindByID", mock.Anything, int64(1)).
        Return(&domain.User{ID: 1, Name: "Orxan"}, nil)

    uc := usecase.NewUserUsecase(mockRepo)
    user, err := uc.GetUser(context.Background(), 1)

    assert.NoError(t, err)
    assert.Equal(t, "Orxan", user.Name)
    mockRepo.AssertExpectations(t)
}
```

### Nümunə 3: sqlc — SQL-dən tip-təhlükəsiz kod

```yaml
# sqlc.yaml
version: "2"
sql:
  - engine: "postgresql"
    queries: "sql/queries/"
    schema: "sql/schema/"
    gen:
      go:
        package: "db"
        out: "internal/db"
        emit_json_tags: true
        emit_prepared_queries: false
```

```sql
-- sql/queries/users.sql

-- name: GetUser :one
SELECT id, name, email, created_at FROM users WHERE id = $1;

-- name: ListUsers :many
SELECT id, name, email FROM users ORDER BY id LIMIT $1 OFFSET $2;

-- name: CreateUser :one
INSERT INTO users (name, email, password_hash)
VALUES ($1, $2, $3)
RETURNING id, name, email, created_at;

-- name: UpdateUserEmail :exec
UPDATE users SET email = $2, updated_at = NOW() WHERE id = $1;

-- name: DeleteUser :exec
DELETE FROM users WHERE id = $1;
```

```go
// generate.go — direktivləri bir yerdə topla
package db

//go:generate sqlc generate
```

```bash
go install github.com/sqlc-dev/sqlc/cmd/sqlc@latest
go generate ./internal/db/
```

```go
// Repository-də istifadə — tam tip-təhlükəsiz
type UserRepository struct {
    queries *db.Queries
}

func (r *UserRepository) FindByID(ctx context.Context, id int64) (*User, error) {
    row, err := r.queries.GetUser(ctx, id)
    if err == sql.ErrNoRows {
        return nil, ErrUserNotFound
    }
    if err != nil {
        return nil, err
    }
    return &User{ID: row.ID, Name: row.Name, Email: row.Email}, nil
}
```

### Nümunə 4: Öz generator yazmaq

```go
// tools/gen/main.go — sadə enum generator
package main

import (
    "flag"
    "fmt"
    "os"
    "strings"
    "text/template"
    "time"
)

var tmpl = template.Must(template.New("").Parse(`// Code generated on {{.Date}}. DO NOT EDIT.

package {{.Package}}

var {{.Name}}Names = map[int]string{
{{- range $i, $v := .Values}}
    {{$i}}: "{{$v}}",
{{- end}}
}

func (s {{.Name}}) Name() string {
    if n, ok := {{.Name}}Names[int(s)]; ok {
        return n
    }
    return "Unknown"
}
`))

func main() {
    pkg := flag.String("package", "", "paket adı")
    name := flag.String("name", "", "tip adı")
    vals := flag.String("values", "", "vergüllə ayrılmış dəyərlər")
    flag.Parse()

    values := strings.Split(*vals, ",")
    
    tmpl.Execute(os.Stdout, map[string]interface{}{
        "Package": *pkg,
        "Name":    *name,
        "Values":  values,
        "Date":    time.Now().Format("2006-01-02"),
    })
}
```

```go
// model.go
//go:generate go run ../../tools/gen/main.go -package=order -name=Status -values=Pending,Paid,Shipped
```

## Praktik Tapşırıqlar

**Tapşırıq 1:**

**Tapşırıq 2:**

**Tapşırıq 3:**

## PHP ilə Müqayisə

Laravel Artisan `make:` komandaları stub faylları yaradır — şablon əsasında. Go-nun `go generate` isə source fayldan tip-təhlükəsiz kod yaradır: interface-dən mock, SQL-dən Go struct.

```php
// Laravel Artisan make: — şablon əsasında fayl yaradır
php artisan make:model User -m        // User.php + migration stub
php artisan make:repository UserRepository  // Stub faylı (əl ilə yazılmış şablon)
php artisan make:request CreateUserRequest  // FormRequest stub

// Nəticə: şablon → sən doldurursan (tip yoxlaması yoxdur)
```

```go
// Go generate — source-dan real kod yaradır
//go:generate mockery --name=UserRepository  // Interface → tam mock implementasiyası
//go:generate sqlc generate                  // SQL → tip-təhlükəsiz Go struct + funksiyalar
//go:generate stringer -type=Status          // iota → String() metodu

// Nəticə: mövcud kod → yeni kod (tip-təhlükəsiz, compile vaxtında yoxlanır)
```

**Əsas fərqlər:**
- Laravel `make:`: boş şablon yaradır, sən doldurursan; Go `generate`: mövcud koddan real implementasiya yaradır
- Laravel stub: runtime-da tip yoxlaması yoxdur; Go generated kod: compile vaxtında yoxlanır
- Laravel: `php artisan` — framework əmri; Go `go generate`: standart tool chain, framework lazım deyil
- `sqlc` vs Eloquent: `sqlc` SQL-dən tip-təhlükəsiz struct yaradır; Eloquent runtime-da sorğu qurur (tip yoxlaması yoxdur)
