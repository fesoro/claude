## Interview Hazırlıq Məsləhəti

**Hər mövzunu mənimsəmək üçün:**
1. Faylı oxu — nəzəri hissəni başa düş.
2. Nümunə kodu əllə yaz (copy-paste etmə).
3. LeetCode-da uyğun məsələləri həll et.
4. Özünü sözlü test et: "Bu alqoritmi namarkere izah edə bilərəm?"
5. Complexity-ni (time + space) sözsüz deyə bilirəmsə — hazıram.

**Bilinməsi vacib olan:**
- Hər data structure-un time/space complexity-si (best/average/worst).
- Nə zaman hansı data structure-u seçmək.
- Trade-off-ları izah etmək bacarığı.
- Edge case-ləri əvvəlcədən qeyd etmək.

## Ümumi Interview Sualları

Aşağıdakı sualları sözlü izah edə bilirsinizmi?

- [ ] Big O Notation-da sabitləri niyə atırıq?
- [ ] Array vs Linked List — nə vaxt hansını seçərdiniz?
- [ ] Hash Map-in worst case niyə O(n)-dir?
- [ ] Binary Search-in loop invariant-ını izah edin.
- [ ] BFS vs DFS — nə vaxt hansını seçərdiniz?
- [ ] Dynamic Programming-in iki şərtini izah edin.
- [ ] Dijkstra-nın niyə negative edge-lərlə işləmədiyini izah edin.
- [ ] Merge sort-un niyə quicksort-dan practikada daha az seçildiyi.
- [ ] Sliding window-da mənfi ədədlər niyə problem yaradır?
- [ ] Topological sort nədir, nə vaxt istifadə olunur?
